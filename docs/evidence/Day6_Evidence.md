# Day6 Evidence - 搜索筛选与体验优化

## 1. 今日完成内容

- 为所有列表页添加了统一的加载状态、空数据和错误提示组件
- 为核心列表页增加了关键词搜索和分类筛选功能
- 优化了发布页的提交反馈：提交中禁用按钮、错误提示
- 抽取了 LoadStatus 和 AdminDeleteBtn 两个复用组件，删除了各视图重复的 CSS

## 2. 列表状态改进

| 状态 | 之前 | 之后 |
|------|------|------|
| 加载中 | 纯文字 "加载中..." | 带旋转动画的 LoadStatus 组件 |
| 空数据 | EmptyState 组件 | LoadStatus 内置空数据展示 |
| 错误 | 无处理 | try/catch 捕获错误，显示错误文案 |
| 筛选 | 无 | 关键词搜索 + 分类/类型下拉筛选 |

## 3. 搜索筛选功能

| 页面 | 搜索方式 | 筛选维度 |
|------|---------|---------|
| TradeView | 关键词搜索（标题+描述） | 分类筛选（6 个分类） |
| LostFoundView | 关键词搜索 | 类型筛选（寻物/拾到） |
| GroupBuyView | 关键词搜索 | 无 |
| ErrandView | 关键词搜索 | 无 |

筛选使用 computed 实时过滤，无需重新请求 API。

## 4. 发布体验优化

- 提交按钮在请求期间显示 "提交中..." 并禁用，防止重复提交
- 请求失败时显示错误提示 "发布失败，请检查 JSON Server 是否运行"
- 提交按钮禁用状态样式：背景变浅 + cursor: not-allowed

## 5. 组件抽取

| 组件 | 用途 | 替换了哪些冗余代码 |
|------|------|-----------------|
| LoadStatus.vue | 统一加载/空/错误状态展示 | 各视图的 `<div v-if="loading">` 模板 |
| AdminDeleteBtn.vue | 管理员删除按钮 | LostFound/GroupBuy/Errand/Trade 4 个视图的重复 CSS |

## 7. 人工调整内容

- 决定将 EmptyState 保留而非完全替换，LoadStatus 的 empty 状态用于简单场景
- 筛选逻辑使用 computed 而非 watch + 新请求，避免不必要的网络请求

## 8. 测试记录

- 关闭 JSON Server 后访问列表页 → 显示 "加载失败：无法获取数据"
- 在 TradeView 搜索框中输入 "机械" → 只显示包含机械键盘的商品
- 选择分类 "教材资料" → 只显示教材类商品
- 在发布页点击提交 → 按钮变为 "提交中..." 并禁用
- 关闭 JSON Server 后提交 → 显示错误提示

## 9. 今日反思

今天的工作聚焦在用户体验优化上。LoadStatus 组件统一了加载/空/错误三种状态的处理，让每个列表页无需重复写条件判断。搜索筛选功能让用户能快速找到感兴趣的内容，是大数据量列表的必备功能。发布页的禁用状态和错误反馈让用户在操作时有清晰的预期，避免重复提交和数据丢失。
